# 最终验收清单

## 一、代码结构验收

- [ ] `room.js` 职责已收敛，不再承担所有视图、动画、数据、弹层逻辑。
- [ ] 低频功能已经按需挂载或分包。
- [ ] 页面中直接写接口 URL 的数量明显减少，集中到 `services/`。
- [ ] 纯函数派生逻辑已抽出：成员网格、关系图、ranking patch。
- [ ] 动画 token 与基础 motion 样式已统一。

## 二、性能验收

- [ ] 不再存在 JS 逐帧 `setData` 飞线动画。
- [ ] 连续 5 次记录脉冲无明显卡顿。
- [ ] 打开数值总览先完成弹层动画，再初始化图表。
- [ ] 弹层关闭后不残留 fixed 遮罩。
- [ ] active room 常驻动画数量明显下降。
- [ ] 图片和头像启用懒加载或缓存。
- [ ] Canvas 海报/镜像卡只在用户明确触发时生成。

## 三、WebSocket 验收

- [ ] WebSocket 握手校验 token。
- [ ] WebSocket 握手校验 room 成员身份。
- [ ] 非成员知道 roomId 也不能订阅房间广播。
- [ ] 有心跳机制。
- [ ] 有指数退避重连。
- [ ] 前后台切换后不会重复连接。
- [ ] 页面卸载后清理监听器和计时器。

## 四、HTTP 与后端权限验收

- [ ] 所有房间读接口校验 `assertRoomMember`。
- [ ] `/storage/presign` 需要登录。
- [ ] 预签名上传限制 MIME、size、objectKey 前缀。
- [ ] `setAllowedOrigins("*")` 已收紧或配置化。
- [ ] `getMyRooms` 不存在明显 N+1 查询。
- [ ] 后端日志包含 requestId。
- [ ] 前端 request 日志包含 duration。
