# Phase 9 Prompt：世界观功能框架统一

## 角色

你是产品架构师 + 前端设计系统工程师。请统一当前微信小程序的“终端 / 驾驶舱 / 脉冲 / 镜像 / 指令 / 航迹”世界观，让功能结构、页面文案和视觉层级一致，同时降低不必要的渲染成本。

## 阶段目标

1. 保留世界观，但从“到处发光”改为“关键路径点亮”。
2. 统一模块命名、状态文案和功能入口。
3. 建立视觉层级和动效预算。
4. 删除过度装饰性区块，提升信息密度和可理解性。
5. 让功能链路更像一个完整系统，而不是多个独立页面拼接。

## 当前世界观理解

项目可以被定义为：

> 一个多人编队协作的“脉冲驾驶舱”系统。用户进入终端，创建或加入编队，在驾驶舱中记录脉冲流转，通过数值总览观察关系，通过指令模块获得每日导航，通过镜像模块生成个人画像，通过身份页管理呼号、头像、音色和偏好。

## 任务一：统一信息架构

底部导航建议只保留核心 3~4 个：

- 驾驶舱
- 指令
- 镜像
- 身份

低频功能放入对应模块内部：

- 历史记录 → 驾驶舱内“航迹日志”
- 结算 → 驾驶舱内“航程封存”
- 音色选择 → 身份页内“声纹模块”
- 等级档案 → 身份页或驾驶舱内“航迹档案”

## 任务二：统一状态文案

建立文案 token：

```js
const copy = {
  cockpitOnline: '驾驶舱已接入',
  cockpitIdle: '等待编队接入',
  roomCreating: '正在建立编队信道',
  roomJoined: '编队信道已同步',
  pulseReady: '脉冲待发射',
  pulseSent: '脉冲已写入航迹',
  archiveConfirm: '确认封存本段航程？',
  mirrorCalibrating: '镜像协议校准中',
  identityCalibrate: '校准呼号'
};
```

要求：

- 同一状态不要多个叫法。
- 重要操作按钮文案要清楚，不要只有抽象世界观词。
- 世界观文案下面可以补一行现实解释。

例如：

```text
航程封存
结算当前编队积分，并生成历史记录
```

## 任务三：视觉层级重排

驾驶舱页优先级：

1. 当前房间状态和成员状态
2. 当前用户可执行操作：记录脉冲、查看总览、封存
3. 最近脉冲记录
4. 次级装饰和背景

删除或下沉：

- 没有实际操作意义的航迹档案块
- 过多占据首屏的装饰 HUD
- 与当前任务无关的长文本说明
- 常驻大型背景动效

## 任务四：动效语义化

动效只用于表达状态变化：

| 状态 | 动效 |
|---|---|
| 接入成功 | 轻微点亮 |
| 编队创建 | 短过渡，不超过 600ms |
| 脉冲记录 | 飞线或目标闪光 |
| 数据总览打开 | 面板滑入 |
| 封存完成 | 成功态短动画 |
| 镜像生成 | loading + 结果显现 |

禁止：

- 无状态变化的常驻炫光。
- 所有卡片同时呼吸。
- 每个页面都用全屏复杂入场动画。

## 任务六：身份页弹窗世界观统一

“校准呼号”弹窗要求：

- 科技感悬浮输入框，但不要过度发光。
- 输入框足够大，适配真机键盘弹起。
- 弹窗内容居中偏上，避免被键盘遮挡。
- 按钮明确：取消 / 确认校准。
- 关闭时卸载弹层。

## 验收标准

- 用户能更清楚知道每个模块做什么。
- 世界观表达统一，不再混乱跳戏。
- 页面视觉更轻，首屏重点更明确。
- 动效数量下降，但关键反馈更明显。
- 真机输入弹窗不再小、不被键盘遮挡。
