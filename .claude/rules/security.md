# 安全规则

## 密钥与日志

- `application.yml` 中的密码和密钥仅用于本地开发，生产环境必须替换。
- JWT secret 至少 256 位。
- 不要提交 `target/`。
- 不要在 console、日志或错误提示中打印 JWT、微信 code、OSS 签名 URL query、连接串或完整 WebSocket URL。
- WebSocket 可以通过 query token 兼容旧客户端，但前端不得打印完整连接 URL。

## 用户可见安全

- 运行时页面、Prompt、备用指令、分享文案、Canvas 海报、Toast、弹窗、按钮、空状态和错误态必须避开审核高风险词。
- 分享卡、Canvas 海报、Toast、弹窗、空状态和错误态都属于运行时用户可见内容，必须经过同一套敏感词和世界观映射。
- 策略和镜像内容必须避开结果预测、结果承诺以及高风险金融/游戏表达。

## 错误处理

- Redis 连接失败、脚本失败和锁失败必须由业务层捕获。
- 返回统一 `Result` 结构和可识别 code。
- 页面只展示简短状态提示。
- 不得向用户暴露堆栈、类名、连接串或敏感 query 值。
