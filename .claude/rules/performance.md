# 性能规则

## 运行预算

- 后端按 2 核 2G 容器预算设计。
- 不要引入高频全空间轮询。
- 不要引入无差别广播。
- 不要在热路径引入阻塞式批量计算。
- 不要反复序列化运行期大对象 JSON。

## Redis 优先

- 当前实现已经把分数、成员元信息、流向事件、待处理本局录和总览缓存放在 Redis。
- 创建、加入、部分记分、生效轮次和归档仍可能访问 MySQL；不要描述成 MySQL 零参与。
- 新增运行期功能优先使用 Redis Hash/ZSet/List 与 Lua 原子操作。
- MySQL 热路径收敛是目标，不是已完成能力。

## 前端运行期

- 页面滚动不能触发 `setData`。
- 触摸绘图必须节流到一帧一次。
- Canvas 组件在不可见或 reduce-motion 生效时必须停止扫描/脉冲动画。
- 长等待 heartbeat 文案建议至少 3 秒轮换一次。
- 分享海报 Canvas 应一次性绘制，不要在滚动或动画中反复重绘。
- 普通卡片避免 blur 和阴影；仅主视觉卡、抽屉、浮层允许少量模糊。

## 实时同步

- WebSocket 必须保持全局单例和稳定回调引用。
- 页面只订阅/取消。
- 不要在普通页面生命周期里销毁并重建连接。
- 调试日志不得打印 JWT、完整 WebSocket URL、微信 code 或 OSS 签名 query。
