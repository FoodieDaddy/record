# 项目概述

## 产品概述

脉冲终端（Smart Record）是一个多人实时协同记录与复盘微信小程序。产品世界观为「脉冲方舟」：用户创建或加入任务编队，在驾驶舱记录一次短程航程中的数值流向；封存后写入航迹档案，航迹样本继续驱动今日指令、镜像投影和身份沉淀。

产品主线保持不变：

```text
编队记录 -> 指令推演 -> 行为画像 -> 身份沉淀
```

用户心智表达：

```text
创建编队，记录脉冲。
展开信标，邀请成员加入。
封存航程，写入航迹档案。
进入导航舱，生成今日指令。
进入全息舱，查看镜像投影。
进入识别舱，管理本舰档案。
```

一句话世界观：

```text
编队是任务，驾驶舱是现场，导航舱是推演，全息舱是映射，识别舱是身份，航迹档案是记忆。
```

## 产品定位

Smart Record 不是普通表单工具，也不是传统娱乐小程序。整体气质是：

```text
黑底、蓝光、克制、冷静、数据感、舰载终端感、复盘感、身份感
```

禁止把产品做成：微信默认表单页、传统设置页、娱乐押注工具、玄学工具、儿童化游戏界面、大红大金视觉、满屏营销卡片、浮夸科幻游戏界面。

## 技术栈

- 后端：Java 21、Spring Boot 3.2.5、MyBatis-Plus、MySQL、Redis/Redisson、WebSocket
- 前端：原生微信小程序
- OSS：后端签发预签名 PUT URL，前端压缩图片后直传
- TTS：Edge-TTS CLI 主引擎、MiMo TTS API 副引擎，ffmpeg 后处理为 44.1kHz 128kbps MP3

## 启动方式

启动基础设施：

```bash
docker-compose up -d
```

启动后端：

```bash
cd backend && JAVA_HOME=$(/usr/libexec/java_home -v 21) mvn spring-boot:run
```

前端：

```text
打开微信开发者工具，导入 miniprogram/ 目录。
```

## 四个主 Tab

| Tab | 舱位定位 | 主要体验 |
|---|---|---|
| 编队 | 驾驶舱 | 创建任务编队、展开信标、加入成员、记录脉冲、封存航程 |
| 指令 | 导航舱 | 读取航迹样本、生成今日指令、查看推进节奏与安全边界 |
| 镜像 | 全息舱 | 校准人格协议、查看镜像投影、协议一致率与全息扫描 |
| 身份 | 识别舱 | 管理本舰呼号、识别徽标、授权等级、通讯/视觉/触感协议 |

## 最终体验目标

用户感知应该是：

```text
我在驾驶舱创建编队。
我展开信标邀请成员加入。
我在导航舱生成今日指令。
我在全息舱查看镜像。
我在识别舱管理本舰身份。
每一次封存，都会写入航迹档案，让系统更懂我。
```

所有页面必须服务于：

```text
记录 -> 复盘 -> 画像 -> 身份
```
