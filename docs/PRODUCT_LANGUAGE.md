# 产品语言

## 核心设定

太空记分器 / Space Scorekeeper（工程代号 smartrecord）—— 深空任务航程记录系统。用户不是在填表，而是在接入个人数据舰船。

产品主线：`任务编队 → 脉冲记录 → 航迹复盘 → 指令推演 → 镜像投影 → 身份沉淀`

核心心智：

- 每个用户 = 一艘个人航船
- 编队 = 一组个人航船为完成同一段短程任务而组成的临时航行单元
- 脉冲 = 任务航程中的核心读数单位，记录推进、链路流向、阶段结果和协作变化
- 封存航程后，所有航迹点写入航迹档案，驱动导航舱、全息舱和识别舱
- 驾驶舱 = 本舰观察编队、写入读数、封存航程的现场（非管理后台）

## 脉冲定义

脉冲 = 任务航程中的核心读数单位。三层含义：

1. **任务读数** — 航船在当前任务中的推进、承载或阶段结果
2. **链路流向** — 读数从哪艘航船释放、向哪艘航船汇入，或主控向全队同步
3. **航迹样本** — 每次脉冲变化成为航迹点，封存后进入航迹档案

**禁止**把脉冲解释为：战斗力、伤害、攻击、输赢结果、押注/筹码、财富/收益、玄学运势。

## 编队定义

编队 = 为完成一段短程任务而临时组成的航行单元。

- 创建编队 = 发起任务航程
- 加入编队 = 接入任务链路
- 编队码 = 6 位短码，只用于手动接入
- 信标 = 邀请、分享、扫码加入等接入入口
- 封存航程 = 正常结束任务并写入航迹档案
- 撤离编队 = 普通成员离开
- 解散编队 = 主控终止当前任务（危险操作，不等于封存航程）

编队不是房间，不是战斗小队，不是社交群。

## 脉冲记录协议

脉冲记录协议 = 不同任务下的脉冲写入方式。所有协议最终都形成脉冲轨迹，封存后写入航迹档案。

| 协议 | 适用场景 | 世界观 | 实现状态 |
|---|---|---|---|
| **脉冲流向** | 成员间互相写入数值流向 | 本舰向目标航船写入脉冲流向，方向 = 读数释放/汇入，数值 = 流向强度 | 已实现（Mode 1，`/score/transfer`） |
| **自航推进** | 每人给自己加，看谁先到目标 | 航船独立推进任务读数，抵达门限 = 完成航段目标 | 未实现 |
| **主控同步** | 主控给全队加同样或不同数值 | 主控向全队写入同步读数，可全队同值或按成员分别写入 | 未实现 |
| **航段写入** | 一轮结束后统一录入 | 航段结束后主控或成员提交本段读数，确认后写入脉冲轨迹 | 已实现（Mode 2，`/round/*`） |

文案禁用：攻击、战斗、对手、击败、输赢、房主发分、管理员加分。

## 驾驶舱设定

- 第一人称本舰视角，透过前方舷窗观察外部编队
- 编队成员 = 外部航船、编队位、HUD 标记和链路读数
- 本舰信息仅作 HUD 小块展示（呼号、脉冲、链路状态、主控身份），不作为头像大卡片出现在舷窗前方
- 航迹档案/回放/全息扫描/历史分析归入全息舱；驾驶舱只保留轻量入口

## 四个主 Tab 舱位定位

| Tab | 舱位 | 主要体验 |
|---|---|---|
| 编队 | 驾驶舱 | 创建编队、信标、加入、写入脉冲、封存航程 |
| 指令 | 导航舱 | 航迹样本、今日指令、推进节奏、安全边界 |
| 镜像 | 全息舱 | 协议校准、全息扫描、镜像投影、协议偏移/演化、航迹档案/回放、镜像图片 |
| 身份 | 识别舱 | 本舰呼号、识别徽标、授权等级、航行经验、装备协议 |

底部 Tab 主文案只显示一行（编队/指令/镜像/身份），选中态通过图标点亮、底板、状态点和能量条表达。

## 封存航程与解散编队

- 封存航程 = 正常结束任务并写入航迹档案，使用主流程按钮
- 解散编队 = 主控终止编队，只作二级危险操作
- 撤离编队 = 普通成员离开
- 红色只用于解散/撤离/失败/错误等危险或负向状态
- 封存航程可用橙色或弱红描边，不做大面积红底

## 页面推荐表达

### 编队页：驾驶舱

优先词：驾驶舱待机、创建任务编队、加入编队、编队码、外部航船、编队位、本舰脉冲、脉冲轨迹、脉冲记录协议、任务读数、航程已封存、样本已写入航迹档案。

信标规则：信标 = 编队邀请与加入入口；编队码 = 6 位短码只用于手动输入/复制；不用「编队码」替代「信标」，不把信标写成「二维码/分享链接」。

协议规则：脉冲流向 → 点击外部航船打开流向面板；自航推进 → 点击本舰推进控件；主控同步 → 主控打开同步面板；航段写入 → 进入航段写入流程。不推荐常驻「记录脉冲」按钮。

封存规则：封存航程用主流程按钮；解散编队只作二级危险操作；撤离编队用于普通成员；不把「封存航程」改成「解散编队」。

视觉强调：第一人称驾驶舱、前方舷窗、外部编队、HUD 仪表、实时感、封存仪式感；避免普通表单、成员卡片列表和管理后台风格。

### 指令页：导航舱

底部 Tab 用「指令」，页面内用「导航舱/导航计算/今日指令」体系。「舰载指令」已收敛为「今日指令」。

生成中文案：航迹协议已同步 / 航迹样本已接入 / 安全边界生成中 / 导航核心校准中 / 指令投影校准中 / 备用导航准备中。

内容规则：只给状态管理、节奏提醒、安全边界和行为复盘视角；禁止预测、玄学、收益承诺或 AI 长文解释。

海报品牌可独立使用 `SPACE SCOREKEEPER`，不附加英文舱位副标题。

### 镜像页：全息舱

页面标题用「全息舱」（非「全息观测舱」）。信息四层：

1. 全息总览 — 镜像投影、协议一致率、航迹样本、全息扫描图
2. 协议行动 — 协议状态卡
3. 协议分析 — 系统判读、信号标签、协议偏移、协议演化（默认折叠）
4. 航迹档案 — 航迹摘要、航迹回放入口

固定操作栏（底部导航上方）：生成镜像图片、重新校准、修改协议。校准流程标题只用「协议校准」。

判读风格：短、冷静、数据感强，不写大段心理鸡汤，不做社交测试娱乐化。

### 身份页：识别舱

优先词：识别舱已接入、本舰档案、本舰呼号、识别徽标、授权等级、航行经验、稳定读数、装备协议、断开终端。

四层架构：①本舰档案（徽标+呼号+授权等级+航行经验）②授权状态（航行经验+稳定读数）③装备协议（通讯/视觉/触感）④终端控制（断开终端）。

航迹数据分析归入全息舱。身份页不做普通「我的」页面，呼号作为本舰铭牌视觉核心，设置项像装备槽位。

## 主操作语言

所有用户可见功能使用中文，不添加英文副标题、英文翻译、英文 kicker 或装饰性英文状态标签，不出现纯英文按钮。主操作用「创建编队 / 加入编队 / 开始导航计算 / 更新协议 / 封存航程 / 撤离编队 / 断开终端」。
