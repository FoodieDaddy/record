# 产品语言

## 核心设定

脉冲终端是一套深空任务记录系统。用户不是在填写表格，而是在接入一艘个人数据舰船。每一次编队创建都是一次短程航程；每一次数值变化都是脉冲轨迹；每一次封存都会写入航迹档案，并让后续的导航核心、镜像系统和档案舱获得更多样本。

产品主线：

```text
编队记录 -> 指令推演 -> 行为画像 -> 身份沉淀
```

核心心智：

```text
编队是任务。
驾驶舱是现场。
导航舱是推演。
全息舱是映射。
识别舱是身份。
航迹档案是记忆。
```

用户心智：

```text
我是一架个人航船。
我可以加入任务编队。
我可以通过信标邀请或加入编队。
编队在驾驶舱中记录脉冲。
指令来自导航舱的航迹推演。
镜像来自全息舱的航迹映射。
身份进入识别舱，管理本舰呼号、识别徽标和装备协议。
```

## 四个主 Tab 的舱位定位

| Tab | 舰船舱位 | 系统身份 | 英文装饰 | 主要体验 |
|---|---|---|---|---|
| 编队 | 驾驶舱 | 任务记录系统 | COCKPIT | 创建编队、展开信标、加入编队、记录脉冲、封存航程 |
| 指令 | 导航舱 | 导航核心系统 | NAV | 读取航迹样本、生成今日指令、校准推进节奏与安全边界 |
| 镜像 | 全息舱 | 行为镜像系统 | HOLO | 校准人格协议、读取航迹样本、展开全息扫描与镜像投影、航迹档案与航迹回放 |
| 身份 | 识别舱 | 身份认证系统 | IDENTITY | 管理本舰呼号、识别徽标、授权等级、航行经验、装备协议 |

底部 Tab 主文案只显示一行（编队/指令/镜像/身份），不显示舱位名和英文装饰。选中态通过图标点亮、底板、状态点和短能量条表达。

## 工程层命名保留

以下工程层命名不在本次产品语言调整范围内，保留不动：

- 数据库表名、字段名（`room`、`all_record`、`dimensions` 等）
- API 路径（`/fortune/today`、`/room/create` 等）
- Redis key
- WebSocket 事件名
- Java 实体和 DTO 字段名
- 前端 JS 变量名和 CSS 类名
- 文件路径（`pages/room/`、`pages/fortune/` 等）

## 页面推荐表达

### 编队页：驾驶舱

优先使用：

```text
驾驶舱待机、创建任务编队、加入编队、编队码、编队席位、记录脉冲、脉冲轨迹、航程已封存、样本已写入航迹档案
```

信标规则：

```text
信标 = 编队邀请与加入入口，可以指二维码、分享入口、扫码加入和加入面板。
编队码 = 6 位短码，只用于手动输入或复制短码。
不要用「编队码」替代「信标」，也不要把信标写成普通「二维码 / 分享链接」。
```

编队页必须强调操作感、实时感、成员席位和封存仪式感，避免普通创建表单和管理后台风格。

避免：

```text
舰员接入、舰员席位（使用「编队成员 / 编队席位」）
```

### 指令页：导航舱

用户可见结构：

```text
导航舱
导航核心
开始导航计算
校准成员状态
接入航迹样本
展开指令投影
今日指令投影
舰载指令
状态读数
推进节奏
安全边界
重新计算
分享指令卡
```

指令页底部 Tab 使用「指令」，页面内部使用「导航舱 / 导航核心 / 指令投影」体系。

指令内容只能给出状态管理、节奏提醒、安全边界和行为复盘视角；禁止做成预测、玄学、收益承诺或 AI 长文解释。

### 镜像页：全息舱

优先使用：

```text
全息舱、镜像舱在线、读取航迹样本、人格协议校准、镜像投影、协议一致率、协议偏移、全息扫描、航迹档案、航迹摘要、航迹回放、镜像尚未稳定
```

镜像判读必须短、冷静、数据感强，不写大段心理鸡汤，不做社交测试娱乐化。

推荐短句风格：

```text
推进倾向偏高。
回稳能力中等。
连续波动后容易出现节奏偏移。
建议降低高频切换。
```

### 身份页：识别舱

优先使用：

```text
识别舱已接入、本舰档案、本舰呼号、识别徽标、授权等级、航行经验、稳定读数、本舰状态、航程徽章、通讯协议、视觉协议、触感协议、装备协议、断开终端
```

识别舱只负责本舰识别、授权等级和装备协议配置。航迹数据分析（航迹档案、航迹回放、净脉冲、样本舱数等）归入全息舱。

身份页不做普通「我的」页面，不做传统设置页。呼号应作为本舰铭牌的视觉核心；设置项应像装备槽位，而不是普通开关列表。

语音相关用户可见表达可以归入身份页「通讯协议 / 通讯音色」。

避免：

```text
舰员代号、舰员档案（使用「本舰呼号 / 本舰档案」）
头盔识别（使用「识别徽标」）
身份等级（使用「授权等级」）
身份档案（使用「识别档案」）
数据矩阵（已移除，使用「本舰状态」）
```

## 主操作语言

所有关键功能必须使用中文。英文只能作为弱装饰、副标题或状态标签：

```text
导航核心          NAV CORE
今日指令投影      FLIGHT DIRECTIVE
镜像投影          MIRROR PROJECTION
本舰档案          SHIP PROFILE
```

不要出现纯英文主按钮。主操作使用「创建编队 / 加入编队 / 开始导航计算 / 更新协议 / 封存航程 / 断开终端」这类中文动词。
