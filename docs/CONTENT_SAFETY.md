# 内容安全

运行时页面、Prompt、备用指令、分享图文、Canvas 海报、Toast、弹窗、按钮、空状态、错误态必须避开审核高风险词。文档可以列出禁词用于约束，但不得把禁词复制到用户可见页面。

## 技术穿帮词禁止直出

以下词可以存在于代码变量、接口字段、日志表或内部注释中，但不得直接出现在运行时用户可见页面、海报、分享标题、按钮、Toast、弹窗正文中：

```text
LLM、LOW-NOISE、MEDIUM-NOISE、HIGH-NOISE、HIGH_RISK、fallback、oracle、fortune、strategy output、THE CALIBRATOR、校准者
```

推荐展示映射：

```text
LLM -> 主引擎
LOW-NOISE -> 低噪
MEDIUM-NOISE -> 中噪
HIGH-NOISE -> 高噪
HIGH_RISK -> 偏高
fallback -> 备用指令
strategy output -> 指令投影
THE CALIBRATOR / 校准者 -> 今日指令 / DIRECTIVE
```

## 审核高风险词

禁止出现在运行时用户可见内容中：

```text
棋牌、赌博、赌、下注、押注、筹码、牌局、牌桌、打牌、麻将、扑克、德州、梭哈、赢钱、赚钱、发财、稳赚、必胜、翻本、追损、运势、算命、占卜、塔罗、抽牌、神谕、卦象、黄历、风水、开运、转运、改运、预测输赢、胜率提升
```

额外禁止直接出现在策略、镜像、分享卡等运行时内容中：

```text
盈利、亏损、收益、胜率、预测、稳赚、暴富、回本、翻盘、赔率、盘口、庄家
```

推荐替换：

```text
任务/回合/场景、积分、结果波动、状态读数、今日指令投影、开始导航计算、指令参考、状态分析、节奏校准、情绪化修正、安全边界、航迹样本
```

## 输出过滤

- 主引擎输出必须通过敏感词过滤。
- 命中敏感词时丢弃生成结果，使用静态备用指令。
- 前端二次展示必须 sanitize，尤其是后端返回的 `title`、`subtitle`、`tags`、`verdict`、poster canvas 文案、分享标题。
- 分享卡、Canvas 海报、Toast、弹窗、空状态、错误态都属于运行时用户可见内容，必须经过同一套敏感词和世界观映射。
- 推荐语气：冷静、克制、短句、舰载终端感、复盘视角。

## 指令与镜像内容边界

- 指令内容只能给出状态管理、节奏提醒、安全边界和行为复盘视角。
- 指令页（导航舱）禁止做成预测、玄学、收益承诺或 AI 长文解释。
- 指令页不得出现「预知」「预测」「运势」「占卜」「神谕」等玄学暗示。
- 镜像判读必须短、冷静、数据感强。
- 镜像页不写大段心理鸡汤，不做社交测试娱乐化。
- 镜像页运行时禁止出现「全息观测舱」，统一为「全息舱」。
- 镜像页运行时禁止出现「黑匣子」「黑匣子样本」，统一为「航迹档案」「航迹样本」。
- 镜像页运行时禁止出现「五维扫描」「五维投影」「五维图谱」「雷达图」，统一为「全息扫描」「全息投影」「全息图谱」「全息扫描图」。
- 镜像页运行时禁止出现「人格测试」，统一为「协议校准」。
- 镜像页运行时禁止出现「行为画像」，统一为「镜像投影」。
- 镜像页 Canvas 海报标识使用「SMART RECORD · HOLO BAY」，不使用「MIRROR PROJECTION」。
- 识别舱运行时禁止出现「舰员档案」「舰员代号」「头盔识别」，统一为「本舰档案」「本舰呼号」「识别徽标」。
- 识别舱运行时禁止出现「数据矩阵」，统一为「授权状态」。
- 识别舱运行时禁止出现「退出登录」「个人中心」「我的」，统一为「断开终端」。
- 识别舱运行时禁止出现「船坞」「船籍舱」「档案舱」。
- 登录页运行时禁止出现「登录」「微信登录」「用户登录」，统一为「接入」「终端接入」「接入终端」。

## 运行时错误展示

- Redis 连接失败、脚本失败、锁失败等服务端错误由业务层统一转换为 `Result` 结构和可识别 code。
- 前端只展示简短状态提示。
- 不展示堆栈、类名、连接串、JWT、微信 code、OSS 签名 query 或完整 WebSocket URL。
