# 镜像页重构设计：从人格终端到全息观测舱

## 概述

将镜像页从「人格终端 / MBTI 测试工具」升级为「全息观测舱 / 行为镜像系统」，实现完整的舱内体验闭环：接入人格协议 → 读取黑匣子样本 → 展开五维扫描 → 生成镜像投影。

### 目标

- 用户进入镜像页时感知到「我进入了全息观测舱」
- 校准流程有扫描仪式感，不是普通弹窗
- 首页信息层级清晰：雷达图居中，协议/判读/信号依次展开
- 分享海报是独立排版的舰载档案卡，不是页面截图
- 所有文案统一到世界观术语体系

### 非目标

- 不改后端 API
- 不改数据模型（profile/stats 接口不变）
- 不改 mbti-const.js、mirror-sanitize.js、mirror-api.js
- 不新增页面文件（原地重构 index.*）

---

## 实现策略

原地重构 `pages/mirror/index.*`，保留现有组件（radar-chart、mbti-swipe-test、mbti-picker-modal），通过 `viewMode` 状态切换实现双视图。

---

## 第一部分：页面状态架构

### 双视图模式

```javascript
data: {
  viewMode: 'main' // 'main' | 'calibration'
}
```

- `main`：观测舱首页（雷达图居中 + 信息层 + 下方区块）
- `calibration`：全屏校准流程（扫描环 + 信号包 + 进度）

两个视图共存于 `index.wxml`，通过 `wx:if` 切换，不使用 navigateTo。数据模型保持不变，`loadProfile()` 和 `loadStats()` 的结果两个视图共享。

### 校准完成流程

1. `mbti-swipe-test` 或 `mbti-picker-modal` 触发 complete/confirm 事件
2. 调用 API 提交
3. 展开协议写入动画（1.5s）
4. `viewMode` 切回 `main`，顶部 toast 显示「协议已同步」

---

## 第二部分：首页布局重构

当前首页是 7 个平铺卡片，重构为三区结构。

### 顶部区：舱位状态

```
全息观测舱              MIRROR BAY
镜像舱在线
```

- 左侧中文标题，右侧弱装饰英文（透明度 0.28）
- 副标题根据状态变化：
  - 未校准 → 「接入人格协议以启动镜像」
  - 已校准 → 「镜像舱在线」
  - 样本不足 → 「黑匣子样本读取中」

### 中部区：镜像核心（雷达图居中）

```
        [雷达图/扫描环 560rpx]
         中央显示 MBTI 类型

   INTJ · 冷静型控场者
   协议一致率 100%
   黑匣子样本 0/3
```

- 雷达图从现有 `radar-chart` 组件扩展，新增 `mode: 'hero'` 居中大尺寸模式
- 未校准时：雷达图锁定态，中央显示「未校准」
- 已校准但样本不足（<3）：雷达图锁定态，中央显示 MBTI 类型，下方显示进度
- 已校准且样本充足：雷达图完整显示五维数据

### 底部区：信息卡片（可滚动）

依次排列，每个卡片用 `sr-card` 样式：

1. **协议状态卡** — 协议编号、来源、同步状态、重新校准/修订按钮
2. **系统判读** — 观察、偏移、风险、成长建议（短句，冷静风格）
3. **信号标签** — 最多 5 个信号标签
4. **协议偏移** — 校准类型 vs 推断类型的对比
5. **协议演化** — 历史校准时间线

底部固定栏按钮从「人格复制」改为「生成镜像卡」。

原来的「任务镜像」和「人格可信度」两个独立卡片不再单独存在——雷达图已包含五维数据，可信度信息合并到协议状态卡中。

## 第三部分：校准流程视图

点击「校准人格协议」后，页面切换到 `calibration` 视图，全屏覆盖。

### 视觉结构

```
┌───────────────────────────┐
│         镜像舱             │
│      协议校准中             │
│                           │
│    ┌─── 扫描环 ───┐        │
│    │   旋转/推进   │        │
│    └──────────────┘        │
│                           │
│      信号 01 / 20          │
│                           │
│   ┌─────────────────┐     │
│   │                 │     │
│   │   题目内容       │     │
│   │                 │     │
│   └─────────────────┘     │
│                           │
│  [不符合]  [跳过]  [符合]   │
│                           │
│         [关闭]             │
└───────────────────────────┘
```

### 交互细节

- 背景暗场，极弱扫描线
- 扫描环复用 radar-chart 的锁定态动画风格，每答一题推进 1/20
- 题卡从右侧淡入，选择后下沉淡出
- 按钮文案：「不符合 / 跳过 / 符合」（替代原来的「否定 / 跳过 / 认同」）
- 进度显示「信号 01 / 20」
- 关闭按钮用 `sr-hatch-close`，点击时弹出 terminal-popup 确认

### 两种校准路径

- 「完整校准」入口 → 进入 calibration 视图，展示 20 题扫描流程
- 「快速接入」入口 → 弹出 `mbti-picker-modal`（保留现有组件），完成后直接回到 main 视图
- 两个入口的按钮文案改为「完整校准」和「快速接入」，各自带一行说明

### 完成动画

20 题完成后：
1. 扫描环闭合（800ms）
2. 中央逐字锁定 MBTI 类型（600ms）
3. 「协议写入中...」文字（400ms）
4. 切回 main 视图，toast「协议已同步」

reduce-motion 下跳过所有动画，直接切回。

## 第四部分：分享海报重绘

Canvas 750×1200，中央核心型布局。

### 布局结构

**顶部**：弱装饰区
- 左上「SMART RECORD」，右上「MIRROR PROJECTION」
- 透明度 0.28，拉开字距

**中央核心**：视觉焦点
- MBTI 类型四字母，字号 72rpx，青色 `--color-cyan`
- 下方类型名（如「冷静型控场者」），字号 32rpx，白色 0.92 透明度
- 外围一圈简化五维扫描线（五条从中心放射的线段，长度对应数值）

**信息区**：核心下方
- 协议一致率 + 百分比
- 黑匣子样本数
- 两行并排，弱蓝边框卡片

**判读区**：短句
- 2-3 句系统判读，字号 26rpx，白色 0.56 透明度
- 不超过 3 行

**底部**：标识区
- 小程序码（右侧）
- 生成时间戳（左侧）
- 底部细线 + 「SMART RECORD · MIRROR PROJECTION」

### 卡片生成动画

保留现有 3 步扫描序列，文案改为：
- `[SCAN] 读取协议参数`
- `[MIRROR] 生成镜像投影`
- `[LOCK] 档案封装完成`

按钮文案改为「保存镜像卡」和「发送给朋友」。

## 第五部分：视觉令牌与动效

### CSS 令牌统一

| 场景 | 令牌 |
|---|---|
| MBTI 类型高亮 | `--color-cyan` |
| 协议已同步 | `--color-green` |
| 样本不足/校准中 | `--color-orange` |
| 正文 | `--text-main` |
| 说明文字 | `--text-secondary` |
| 装饰英文 | `--text-muted` |
| 卡片边框 | `rgba(10,132,255,0.18)` |
| HUD 面板 | `--sr-hud-panel` 样式 |

### 动效守卫

所有新增动画必须：
- 绑定 `animationEnabled` 判断
- 页面根节点绑定 `reduce-motion` 类
- `onHide`/`onUnload` 清理所有 timer/interval/animation frame
- 雷达图组件已有 `reduceMotion` prop，确保传入

### 页面进入动效（main 视图）

```
0ms    背景暗场
120ms  顶部标题淡入
240ms  雷达图核心点亮
420ms  五维扫描线展开
600ms  协议卡和按钮浮现
```

reduce-motion 下全部跳过，直接显示最终状态。

## 执行顺序

1. **文案替换** — 全局搜索替换旧词，更新 `index.wxml` 中所有可见文案
2. **首页布局重构** — 重写 `index.wxml` 三区结构，调整 `index.wxss`
3. **数据模型调整** — `index.js` 中合并可信度到协议状态卡，新增 `viewMode` 状态
4. **校准流程** — 实现 calibration 视图，修改 `mbti-swipe-test` 按钮文案
5. **分享海报** — 重写 `_drawPersonaCard()` Canvas 绘制逻辑
6. **动效与守卫** — 添加页面进入动画、reduce-motion 绑定、timer 清理
7. **验收** — 检查穿帮词、高风险词、英文比例、红色使用、底部安全区

---

## 验收标准

用户感知：

```
我进入了全息观测舱。
我正在校准人格协议。
系统读取黑匣子样本。
五维扫描正在生成镜像投影。
我可以把镜像投影保存为一张独立的舰载档案卡。
```

检查项：

- [ ] 所有主操作为中文，英文仅弱装饰
- [ ] 运行时不出现审核高风险词
- [ ] 运行时不出现技术穿帮词（LLM、LOW-NOISE、HIGH_RISK、fallback 等）
- [ ] 红色只用于危险/错误/失败
- [ ] 页面根节点绑定 reduce-motion
- [ ] 所有 timer/interval/animation frame 在 onHide/onUnload 清理
- [ ] 雷达图 reduceMotion prop 正确传入
- [ ] 分享海报独立排版，文字清晰可读
- [ ] 镜像判读短、冷静、数据感强
- [ ] 底部安全区不遮挡按钮
