# 脉冲方舟世界观 — 全局术语与文案统一设计

## 概述

将 Smart Record 全系统的用户可见文案统一为「脉冲方舟」舰载终端风格。不改页面结构、不改视觉布局、不新增功能，只改文案和术语。覆盖前端 WXML/JS 和后端 BizException message。

## 实施路径

术语表建基线 → 逐页逐模块扫描替换 → 上下文校准。

## 1. 术语映射表

全局基线，所有前后端用户可见文案按此表替换。

| 原词 | 替换为 | 适用范围 |
|------|--------|---------|
| 房间 | 空间 | 全局 |
| 房间号 | 识别码 | 全局 |
| 创建房间/创建空间 | 启动空间 | 前端+后端 |
| 加入房间/加入空间 | 接入空间 | 前端+后端 |
| 房间不存在 | 空间不存在 | 后端 BizException |
| 房间已关闭 | 空间已关闭 | 后端 BizException |
| 房间已结算 | 空间已封存 | 后端 BizException |
| 房间人数已达上限 | 空间舰员已满 | 后端 BizException |
| 成员 | 舰员 | 前端 UI 文案 |
| 记分/记录 | 记录脉冲 | 前端操作文案 |
| 流水/流水记录 | 脉冲轨迹 | 前端 UI |
| 结算 | 封存 | 前端+后端 |
| 结算成功 | 航程已封存 | 前端 toast |
| 历史记录 | 黑匣子 | 前端 UI |
| 今日策略 | 今日航行指令 | 前端 UI |
| 生成策略 | 启动策略核心 | 前端按钮 |
| 重新生成 | 重新点火 | 前端按钮 |
| 生成中 | 核心启动中 | 前端状态 |
| 人格测试 | 协议校准 | 前端 UI |
| 行为画像 | 镜像投影 | 前端 UI |
| 设置 | 装备协议 | 前端 UI |
| 退出登录 | 断开终端 | 前端按钮 |
| 昵称 | 舰员代号 | 前端 UI |
| 头像 | 头盔识别 | 前端 UI |

**不动的内容**：底部 Tab 名（空间/策略/镜像/身份）、数据库字段名、API 路径、Redis key、DTO @Schema 注解、后端日志。

## 2. 前端文案改造

### 2.1 空间页 (pages/room)

**WXML 静态文本**：
- `尚未登录` → `终端未接入`
- `登录后即可启动或接入空间` → `接入终端后可启动或接入任务空间`
- `去登录` → `接入终端`

**JS toast/modal**：
- `创建失败` → `启动失败`
- `加入失败` → `接入失败`
- `记录成功` → `脉冲已记录`
- `录入成功` → `录入已提交`
- `确认退出？` → `确认退出当前空间？`
- `正在归档...` → `正在封存航程...`
- `暂无可封存数据` → `暂无可封存的航程数据`
- `已退出` → `已断开`

### 2.2 策略页 (pages/fortune)

- `今日策略` → `今日航行指令`
- `核心已复位 · 点击卡牌启动新一轮策略推演` → `策略核心待机 · 点击启动`

- `核心洞察` → `当前状态`
- `行动优势` → `执行节奏`

**JS**：
- `策略生成失败` → `策略核心启动失败`
- `生成失败` → `启动失败`

### 2.3 镜像页 (pages/mirror)

**WXML**：

**JS**：
- `镜像加载失败` → `镜像投影加载失败`
- `同步失败` → `协议同步失败`
- `生成失败` → `投影生成失败`
- `档案图已保存` → `镜像档案已保存`

### 2.4 身份页 (pages/profile)

### 2.5 结算页 (pages/settle)

**WXML**：
- `正在生成档案卡...` → `正在生成航程档案...`

### 2.7 脉冲日志页 (pages/score-records)

## 3. 后端文案改造

### 3.1 BizException 消息

**RoomServiceImpl**：
- `你已有活跃房间，请先退出后再创建` → `你已有活跃空间，请先退出后再启动`
- `请输入房间号` → `请输入识别码`
- `房间不存在` → `空间不存在`
- `房间已关闭` → `空间已关闭`
- `房间人数已达上限，无法加入（最多16人）` → `空间舰员已满（上限16人）`
- `用户不存在，请重新登录` → `身份未识别，请重新接入终端`
- `仅房主可解散房间` → `仅主控可解散空间`
- `房间已结算，不能修改记分设置` → `空间已封存，不能修改记录设置`

**IdentityLevelServiceImpl**：
- `用户不存在` → `身份未识别`

**MirrorProfileServiceImpl**：
- `必须提交20题答案` → `协议校准需要完成20题`
- `非法MBTI类型编号` → `协议类型无效`

**OverviewServiceImpl**：
- `房间不存在` → `空间不存在`

**TtsController**：
- `TTS 合成失败` → `语音合成失败`

### 3.2 不改动的内容

- 后端日志（log.warn / log.error）：面向开发者，保持当前中文措辞
- DTO @Schema 注解：面向 Swagger 文档，暂不改动
- API 路径、数据库字段名、Redis key

## 4. 黑匣子集成

黑匣子是对现有"历史记录/结算归档"概念的舰载化包装，不是新功能。

### 4.1 结算流程

结算成功后的 toast：
- `航程已封存`
- settle 页面结算完成状态加一行：`样本已写入黑匣子`

### 4.2 身份页数据矩阵

- `任务数` → `样本数`
- sub 文字 → `黑匣子样本`
- 数据来源：复用现有 `matchCount`，不新增接口

### 4.3 黑匣子概念出现场景

- 结算成功提示：`航程已封存，样本已写入黑匣子`
- 身份页数据矩阵：`样本数 / 黑匣子样本`
- 镜像页样本不足：`黑匣子样本不足，完成 3 次航程封存后解锁`
- 策略页生成说明：`同步身份协议与黑匣子样本`

### 4.4 不出现的场景

- 空间页实时记录中不提及黑匣子
- 登录页不提及
- 普通错误提示中不提及

## 5. 实施顺序

1. **后端 BizException 消息**（RoomServiceImpl、其他 Service、Controller）
2. **前端空间页**（room.wxml + room.js）
3. **前端策略页**（fortune.wxml + fortune.js）
4. **前端镜像页**（mirror/index.wxml + mirror/index.js）
5. **前端身份页**（profile.wxml + profile.js）
6. **前端结算页**（settle.wxml + settle.js）
7. **前端登录页**（login.wxml + login.js）
8. **前端其他页面/组件**（score-records、settings、voice-select、各组件）
9. **全局检查**：grep 扫描残留的旧术语

## 6. 验收标准

- 用户可见文案中不再出现"房间"、"成员"、"结算"、"生成策略"、"重新生成"、"设置"、"退出登录"、"登录"（作为按钮/标题）等旧术语
- 后端 BizException 消息中不再出现"房间"
- 黑匣子概念在结算、身份、镜像、策略四处正确出现
- 底部 Tab 名不变
- API 路径、数据库字段、Redis key 不变
- 后端日志不变
