# Room Terminal 全面改造设计

**日期**: 2026-06-06
**范围**: 房间页 (room.js/wxml/wxss) + 相关组件 + 结算弹层
**目标**: 将房间页从"记分工具"重构为"积分流转终端"

---

## 设计决策

| 决策点 | 结论 |
|---|---|
| 成员点击交互 | 方案 A：单击目标 → 弹出终端 (FROM 固定为自己) |
| Mode 2 改造 | 本局录入 UI 也统一改造 |
| 积分关系图 | 从房间页移除，仅保留结算报告中的关系图 |
| 结算弹层 | 按新风格统一改造 |

---

## 一、页面结构

房间页保持 5 个视图状态不变，仅改造「在房间中」和「结算弹层」两个状态的视觉和交互。

在房间中的模块顺序：

```
ROOM TERMINAL          ← 顶部终端栏（改造）
PLAYER NETWORK         ← 成员网络（改造）
FLOW LOG               ← 积分流（改造）
ROOM INSIGHT           ← 战局洞察（改造）
SETTLE & EXIT          ← 结算按钮（改造）
```

移除：NETWORK MAP (积分关系图) 从房间页移除。

---

## 二、顶部终端栏 (ROOM TERMINAL)

### 当前问题
信息层级不够分明，操作按钮风格不统一。

### 目标结构
```
ROOM TERMINAL                              ● ONLINE

ROOM ID
X23G43

MODE
自由流转

PLAYERS
2 / 16

[复制] [邀请] [二维码]
```

### 视觉规范
- 背景: `#0A0F18`
- 边框: `rgba(0,170,255,0.15)`
- 圆角: `16px`
- 主色: `#00AFFF`
- 在线状态: `#36FF74`
- 次级文字: `#7C8698`

### 操作按钮
复制、邀请、二维码按钮统一为 HUD 图标按钮：
- 宽高 `36px`
- 透明背景
- 蓝色弱描边
- 点击时蓝色发光

## 三、成员网络 (PLAYER NETWORK)

### 成员节点结构
```
[头像 + 赛博环]
昵称
积分
```

### 头像环状态
- **普通状态**: 灰蓝细环 (`rgba(255,255,255,0.12)`)
- **选中状态**: 蓝色能量环 + 轻微呼吸光 (`#00AFFF`)
- **正收益玩家**: 绿色弱光环 (`#36FF74`)
- **负收益玩家**: 红色弱光环 (`#FF4D4F`)
- **房主**: 头像右上角 HOST 小标签

### 头像尺寸
- 头像: `72rpx - 88rpx`
- 外环: 头像 + `8rpx`

### 昵称
- 最多 5-6 个中文字符，超出省略号
- 颜色: `#FFFFFF` 或 `#BFC7D5`

### 积分显示
- 正数: `#36FF74`，带 `+` 前缀
- 负数: `#FF4D4F`，带 `-` 前缀
- 零分: `#7C8698`

### 布局规则
- 人数 ≤ 4: 横向节点布局
- 人数 > 4: 横向滚动
- 最多一行，不撑高

### 点击交互（方案 A）
1. 点击目标成员
2. FROM 自动设为自己
3. 弹出流转终端

### 选中反馈
- 头像外环变蓝
- 下方显示 `TARGET LOCKED`
- 卡片轻微上浮

### 数字键盘样式
- 按钮背景: `rgba(255,255,255,0.04)`
- 按钮边框: `rgba(0,170,255,0.12)`
- 点击: 蓝色短闪
- 数字: 白色
- 功能键: 灰色
- 确认键: `#00AFFF`

---

## 五、积分动画规则

### 主动画：积分流光
从扣分玩家头像 → 加分玩家头像：
- 蓝色/绿色能量光点沿路径飞行
- 携带 `+66` 或 `-66` 小数字
- 持续 `600ms - 900ms`
- 颜色: 加分 `#36FF74`，扣分 `#FF4D4F`，轨迹 `#00AFFF`

### 辅助动画：数字跳变
- 原数字淡出，新数字淡入
- 轻微 scale `1.0 → 1.12 → 1.0`
- 持续 `300ms`

### 禁止
- 不要同时长时间滚动数字 + 飞行动画
- 不要多条动画重叠导致卡顿
- 只对参与本次流转的两名成员播放

## 六、积分流 (FLOW LOG)

### 布局
```
FLOW LOG
积分流

04:00
● 硬核的经验包      6 →      摆烂的铁公鸡
● 摆烂的铁公鸡      5 →      硬核的经验包
```

### 视觉规则
- 时间: `#7C8698`
- 来源玩家: `#BFC7D5`
- 积分值: `#00AFFF` 或根据流向正负
- 箭头: `#00AFFF`
- 目标玩家: `#FFFFFF`

### 我参与的记录
- 左侧蓝点高亮
- 整行背景 `rgba(0,170,255,0.05)`

### 空状态
```
FLOW LOG
积分流

NO FLOW DATA
暂无积分流

完成一次积分流转后，系统将在此记录数据。
```
加一个微弱脉冲点。

## 八、结算按钮 (SETTLE & EXIT)

### 样式
- 蓝色描边
- 透明背景
- 主文字 `#00AFFF`
- 副标题 `#7C8698`
- 点击时蓝色扫描光经过

### 二次确认
结算不可逆，点击后弹出确认：
```
SYSTEM WARNING
确认结算并退出房间？

结算后本局积分将写入档案。

[取消] [确认结算]
```
确认按钮使用蓝色，不用红色。

---

## 九、结算弹层 (SETTLE OVERLAY)

统一按赛博终端风格改造，保持现有 6 段结构不变：

1. **战局总结** (battle-summary) — 统一配色
2. **积分趋势图** (score-chart) — 曲线颜色统一为蓝绿红体系，移除橙色
3. **积分关系网络** (score-network) — 保留（仅结算时展示）
4. **战局洞察** (battle-insight) — 统一数据块风格
5. **最终排名** — 统一配色和间距
6. **人格信号** (persona-signal) — 统一风格

### 趋势图配色
- 正收益用户曲线: `#36FF74`
- 负收益用户曲线: `#FF4D4F`
- 网格线: `rgba(255,255,255,0.05)`
- 移除橙色

## 十、本局录入 (Mode 2) 改造

### 录入本局按钮
改为赛博终端风格按钮，与结算按钮统一。

### 轮次状态条
保持现有组件结构，统一配色为蓝光体系。

### 弹窗（房主填写 / 成员自填 / 确认）
统一改造为暗色玻璃卡风格：
- 背景: `rgba(10,15,24,0.95)`
- 边框: `rgba(0,170,255,0.15)`
- 标题: 赛博 kicker 风格
- 按钮: 统一 HUD 风格

## 十一、字体与文字统一

### 模块英文标题
```
ROOM TERMINAL / PLAYER NETWORK / FLOW LOG / ROOM INSIGHT
```
样式：`#00AFFF`, `letter-spacing: 4rpx`, `font-size: 20rpx`

### 中文标题
样式：`#FFFFFF`, `font-weight: 600`

### 说明文字
`#7C8698`

### 重要数字
使用更大字号，正负色明确。

---

## 十二、WXSS 规范

- 不使用 `*` 通配符（WXSS 不支持）
- 显式列出类名
- `transition` 不用 `all`，显式指定属性
- 动画遵循 `reduce-motion` 全局开关
- 背景统一 `#0A0A0A`，可使用极弱径向光

---

## 十三、验收标准

1. 成员头像有统一赛博头像环（普通/选中/正收益/负收益/房主）
2. 正负积分颜色统一，不再像普通文本
3. 积分流列表改为时间线 / 数据流布局
4. 给分动画只保留一套主动画（流光飞行），数字使用短促跳变
5. 积分总览弹窗使用全局蓝绿红配色，不使用橙色曲线
6. 战局洞察改为数据块，不再像普通列表
7. 结算并退出按钮有二次确认弹窗
8. 所有按钮、图标、文字符合 Cyber Terminal 风格
9. Mode 2 弹窗统一改造为暗色玻璃卡风格
10. 结算弹层 6 段结构视觉统一
11. 房间页移除积分关系图
12. 页面整体看起来像积分流转终端
