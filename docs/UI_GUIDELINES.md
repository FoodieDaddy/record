# UI 指南

## 视觉系统

统一黑底，允许极弱径向光：

```css
background:
  radial-gradient(circle at 20% 0%, rgba(10,132,255,0.12), transparent 32%),
  radial-gradient(circle at 90% 18%, rgba(94,92,230,0.08), transparent 30%),
  #0A0A0A;
```

主色令牌：

```css
--color-primary: #0A84FF;
--color-cyan: #00C8FF;
--color-purple: #5E5CE6;
--color-green: #30D158;
--color-orange: #FF9F0A;
--color-red: #FF453A;
--text-main: rgba(255,255,255,0.92);
--text-secondary: rgba(255,255,255,0.56);
--text-muted: rgba(255,255,255,0.38);
--text-disabled: rgba(255,255,255,0.24);
```

- 蓝色/青色只做主高亮、数据焦点、航行核心、指令投影和按钮焦点。
- 绿色只表达在线、已连接、同步完成等正向系统状态。
- 橙色只表达偏高、注意、边界、校准中。
- 红色只用于危险、退出、错误、失败，禁止用于分享按钮、普通状态 badge、非危险提示。
- 正文透明度不要低于 `0.40`；装饰性英文可低到 `0.24-0.28`，但不能承载关键含义。
- 普通页面不要大面积纯蓝、大面积渐变、全屏发光或单色系铺满。
- 旧 `.glass-card` 可以保留兼容，但新 UI 优先使用 `sr-*` 终端令牌；普通卡片不要全部 `backdrop-filter` 或全部 `box-shadow`。

推荐卡片：

```css
.sr-card {
  border-radius: 28rpx;
  border: 1rpx solid rgba(10,132,255,0.18);
  background: rgba(255,255,255,0.035);
}

.sr-card-secondary {
  border-radius: 24rpx;
  border: 1rpx solid rgba(255,255,255,0.08);
  background: rgba(255,255,255,0.025);
}
```

舰载 HUD 核心框推荐：

```css
.sr-hud-panel {
  border: 1rpx solid rgba(0,200,255,0.22);
  background: rgba(3,10,18,0.64);
  clip-path: polygon(
    24rpx 0,
    calc(100% - 24rpx) 0,
    100% 24rpx,
    100% calc(100% - 24rpx),
    calc(100% - 24rpx) 100%,
    24rpx 100%,
    0 calc(100% - 24rpx),
    0 24rpx
  );
}
```

## 终端风格

舰载终端感来自克制层级，不来自堆砌发光元素。

允许：细描边、切角按钮、弱扫描线、HUD 小标签、状态点、数据条、甲板透视线、全息扫描环、极少量英文 kicker、数字等宽。

禁止：大面积发光、所有文字都变蓝、每个按钮都有光晕、持续粒子、控制台日志堆砌、大量英文缩写、AI 长段解释、模拟游戏战斗 UI。

中文标题字距要克制，不要过度拉开导致可读性下降；英文装饰可以拉开字距。

## 底部系统 Dock

底部导航使用「舰载系统 Dock」风格，不是普通微信 TabBar。

### 四个舱位插槽

| Tab | 舱位 | 英文装饰 | 图标隐喻 |
|---|---|---|---|
| 编队 | 驾驶舱 | COCKPIT | HUD 框 + 舰点 + 连接线 |
| 指令 | 导航舱 | NAV | 轨道环 + 核心点 + 航线 |
| 镜像 | 全息舱 | HOLO | 菱形投影框 + 扫描线 |
| 身份 | 识别舱 | IDENTITY | 四角扫描框 + 中心识别点 + 信息线 |

### Dock 容器规范

- 固定底部，适配 `env(safe-area-inset-bottom)`
- 基础高度 140rpx，叠加安全区
- 背景：`linear-gradient(180deg, rgba(4,12,24,0.98), rgba(0,0,0,0.98))`
- 顶部弱蓝线：`border-top: 1rpx solid rgba(10,132,255,0.18)`
- 中央能量轨：宽度 65%，从透明到青蓝再到透明
- 不做大面积发光、不过度玻璃拟态

### 舱位插槽规范

- 未选中图标：`border-color: rgba(255,255,255,0.44)`，文字 `rgba(255,255,255,0.55)`
- 选中图标：`border-color: rgba(0,200,255,0.88)`，高亮点 `#00C8FF`
- 选中底板：`background: rgba(10,132,255,0.08)`
- 选中显示舱位名和英文装饰
- 状态点：`#00C8FF` + 弱光晕
- 底部能量条：青蓝渐变

### 图标实现要求

- 纯 CSS/WXML 结构，不使用 image、远程图片、Emoji
- 四个图标必须有明确形状差异，遮住文字也能区分
- 选中态像「舱位已接入」，不只是颜色变化
- 不使用 `transition: all`，仅 `opacity`、`border-color`、`background-color`、`transform`
- reduce-motion 兜底：`transition: none !important`

## active 全局星空背景规范

- 编队页 active 首屏使用全局星空背景，不再使用 `cockpit-active.webp` 作为主背景。
- 背景由本地 CSS 渐变、弱星点、深蓝光晕和低亮网格组成，保持与全局黑底主题一致。
- 动态覆盖层必须保持可点击、可更新，外部航船 marker 仍由 WXML 生成并放在舷窗上半区。
- 推荐比例为 `690:940`，页面主容器保持 `calc(100% - 56rpx)` 宽度与底部安全间距，避免被 Dock 挡住。
- 覆盖层定位优先用百分比，不依赖图片内硬编码文案，也不回到多张卡片拼接。
- 不引入远程图片、GIF、视频或高频动画；星空背景必须能在 reduce-motion 下保持静态。

## 页面分工

### 编队页（驾驶舱）

负责创建编队、加入编队、编队配置、成员布局、数值记录、实时流水、封存归档。文案使用「编队」「编队码」「信标」「加入」「编队席位」「记录脉冲」「封存」「航迹档案」，避免普通表单感。

驾驶舱主视觉规范：

- 驾驶舱是一体化外壳（cockpit-shell），舷窗区域（cockpit-window）和驾驶台区域（cockpit-deck）构成一个连续空间，不是两张卡片叠放。外壳使用八角切角（clip-path polygon），驾驶台向上偏移 40rpx 与舷窗底部融合。
- 用户当前视角就是本舰第一人称，本舰不得作为头像大卡片出现在主视觉中心。
- 本舰脉冲使用半圆仪表盘（外弧 + 内弧 + 11 根刻度 + 标签 + 数值 + 状态 + 仪表底座），嵌入驾驶台顶部，向上偏移 90rpx 与舷窗底部融合。不使用完整圆环悬浮在画面中央。
- 编队成员应表现为飞船目标标记（ship-target）：飞船剪影（ship-silhouette）包含三角机鼻 + 双翼条 + 核心识别徽标 + 护盾环 + 链路灯，下方 HUD 标签（呼号 + 脉冲值）。不使用头像大图、Emoji、远程图片、大图标或球形包裹。
- 舷窗角落只保留轻量数值读数（编队码、成员数、模式、链路），底部角落读数使用更低透明度（vch-value--faint），减少与下方信息区的信息重复。
- 终端屏幕（terminal-screen）与半圆仪表通过中轴线（deck-spine）连接，共用驾驶台深色底。终端包含编队信息行（编队码+复制+芯片）、实时脉冲行（标题+徽章）、最近流向和脉冲轨迹预览行（sparkline+操作按钮）。
- 编队码、编队成员数、记录模式、航程阶段、链路状态在终端屏幕内的 terminal-meta-row 和 terminal-chip-row 中集中展示，不做普通信息卡片。
- 航迹档案在驾驶舱只保留轻量入口，引导进入全息舱查看航迹映射；驾驶舱不展示完整历史分析、航迹回放列表或全息扫描。

active 态全局星空规范：

- 首屏使用 `cockpit-main` 包裹 `cockpit-starry-shell`，不在其下方继续堆叠大型编队信息卡、实时脉冲卡或轨迹卡。
- `cockpit-starry-shell` 使用 `690:940` 比例，内部背景层为 `cockpit-starry-bg` / `cockpit-starry-nebula` / `cockpit-starry-grid` / `cockpit-starry-vignette`。
- 本舰仪表使用 `overlay-gauge`，宽约 340rpx，高约 144rpx，数值字号控制在 54rpx-64rpx。
- 终端屏使用 `overlay-terminal`，集中展示编队码、复制、成员数、模式、阶段、实时脉冲、最近流向和 `trajectory-mini` 点阵预览。
- 外部航船使用 `ship-craft`：机鼻、左右翼、中心识别徽标、护盾环和在线点必须真实存在；头像只作为 34rpx-40rpx 的识别徽标，不作为头像圆环主视觉。
- 外部航船 1-4 艘时使用固定坐标：1 艘 `(50,34)`；2 艘 `(34,36)/(68,28)`；3 艘 `(26,38)/(50,26)/(74,36)`；4 艘 `(24,38)/(44,24)/(64,26)/(82,38)`；y 坐标不得超过 44%，避免贴近控制区。
- 轨迹预览有数据时生成 5-8 个点，x 均匀分布，y 归一到 18%-82%；无数据时显示固定 5 点空态和「等待更多脉冲写入」，不得留下空白框。
- `overlay-control` 放在星空舞台底部，至少完整露出「记录脉冲 / 展开信标」两个主按钮，底部 Dock 不得遮挡。

编队页信标规则：

- 顶部或分享区的二维码、分享入口、扫码加入统一称为「信标」。
- 6 位短码称为「编队码」，只用于手动输入或复制短码。
- 主操作可以使用「展开加入信标 / 发送信标 / 保存信标 / 扫描信标」。
- 不要把信标退回成普通「二维码」「分享链接」或全部替换为「编队码」。

视觉关键词：第一人称驾驶舱、一体化外壳、上窗下台、前方舷窗、深空外景、飞船剪影标记、半圆脉冲仪表、终端屏幕、脉冲轨迹预览、信标控制键、航程控制、封存航迹档案。

### 指令页（导航舱）

负责今日状态、行动建议、安全边界、指令卡分享的产品功能，但用户可见体验必须包装为「导航舱 / 今日指令投影」。展示内容必须是复盘建议、节奏提醒、状态管理、安全边界，不做结果预测，不承诺收益，不引导冒进。

视觉关键词：甲板透视线、导航核心、星图环、航线节点、计算动画、指令投影、系统日志舱。

交互原则：

- 页面标题为「导航舱」，英文 kicker 为「NAV BAY ONLINE」。
- 主按钮为「开始导航计算」，kicker 为「PRESS TO CALC」。
- 点击计算后，动画要从当前导航核心原地启动，避免突然黑屏或像普通 loading。
- 生成中导航核心必须保持可见，不能出现长时间空黑屏。
- 生成中日志使用「航迹协议已同步 / 航迹样本已接入 / 安全边界生成中」。
- 生成中长等待文案使用「导航核心校准中 / 指令投影校准中 / 备用导航准备中」。
- 结果页优先展示「今日指令」，再展示状态读数、推进节奏和安全边界。
- 重新计算确认后必须回到 launch 待机页，由用户再次点击「开始导航计算」，不要自动生成。
- 分享卡必须是独立海报排版，不要把页面截图缩小塞进预览。
- 海报标识为「SMART RECORD · NAV BAY」。
- 底部状态使用「航迹协议」，不使用「成员协议」。

### 镜像页（全息舱）

负责 MBTI 协议、历史表现画像、人格一致性、全息扫描、航迹档案、航迹回放、画像分享。镜像判读必须短、冷静、数据感，不写大段 AI 心理鸡汤。

视觉关键词：全息投影、扫描环、人格协议、协议偏移、全息扫描、航迹样本读取、航迹档案。

### 身份页（识别舱）

负责本舰身份识别、授权等级、航行经验、稳定读数、装备协议（通讯/视觉/触感）和断开终端。航迹数据分析归入全息舱。

信息架构分为四层：

```text
一级：本舰档案 — 识别徽标、本舰呼号、授权等级、航行经验
二级：授权状态 — 航行经验、稳定读数
三级：装备协议 — 通讯协议、通讯音色、视觉协议、触感协议
四级：终端控制 — 断开终端
```

视觉关键词：本舰铭牌、识别徽标、授权等级、授权状态、装备槽位、通讯/视觉/触感协议、终端控制。

底部安全区规范：

- 根容器 `padding-bottom` 至少 `calc(200rpx + env(safe-area-inset-bottom))`，确保「断开终端」按钮完整显示在自定义 tabbar 上方。
- 「断开终端」使用流式布局，不使用 fixed/sticky，避免与 tabbar 层叠冲突。
- 「断开终端」使用透明底 + 红色细描边，不做大红底按钮。
- 退出确认弹窗（wx.showModal）按钮完整显示，不被 Home Indicator 遮挡。

呼号校准弹窗规范：

- 使用居中悬浮 overlay，不使用底部抽屉。
- overlay z-index 高于 custom-tab-bar（当前 1000 vs 999）。
- 面板背景深黑蓝半透明 + 青蓝细描边，保持终端风格。
- 输入框使用 `type="text"`，不使用 `type="nickname"`，避免触发微信白色昵称建议条。
- 键盘弹出时面板通过 `transform` 上移，确保输入框和按钮可见。
- 按钮文案：「写入呼号」（主操作）、「取消」（次操作）。
- 不出现「昵称」「用户名」等旧词。

登录页（终端接入）底部安全区规范：

- 登录页无 tabbar，但底部信息区域必须考虑 Home Indicator。
- `.bottom-info` 使用 `bottom: calc(60rpx + env(safe-area-inset-bottom))`。
- 接入中卡片垂直居中，不与底部区域冲突。

## 组件与布局规范

- 按钮：主按钮高度 72rpx-88rpx，不超过 96rpx。优先细描边、切角、状态点，不做传统大圆角纯蓝按钮。
- 异步按钮：必须使用「文本绝对居中 + 图标/Loading 绝对定位」结构，Loading 状态文字位置不能抖动。
- 危险操作：退出、删除、解散、断开终端使用透明底 + 红色细描边，不做大红底按钮。
- 分享/保存操作：分享、保存、发送给朋友使用蓝色/青色，不使用红色。
- 线框图标：全站 WXML/WXSS/运行时数据禁止原生彩色 Emoji。图标使用纯色线框、CSS icon、SVG path 或图标字体。
- 成员简易模式：16 人以内使用 4x4 矩阵网格，配合 `scroll-view` 和 `max-height: 600rpx`，大数字必须格式化并防溢出。
- 座位模式：使用绝对定位舞台，中央仅作为空间参考点，成员通过 `pos-top` / `pos-bottom` / `pos-left` / `pos-right` 环绕排布。
- 弹窗：用 `wx:if` 懒渲染；底部抽屉、确认弹窗、选择器必须支持 reduce-motion 的静默展开。
- 底部安全区：Tab 页面内容区必须考虑自定义 tabbar 与 `env(safe-area-inset-bottom)`，结果页、长列表、底部按钮至少预留 180rpx-240rpx。
- 海报预览：分享海报不是页面缩略图，必须做独立排版，保证主指令/核心信息可读。

## 动效静默管理

- 全局开关为 `animationEnabled`，状态来自 `app.globalData` 与 Storage。
- 核心页面根节点必须绑定 `reduce-motion`：

```xml
<view class="page-container {{!animationEnabled ? 'reduce-motion' : ''}}">
```

- `app.wxss` 必须提供全局兜底：

```css
.reduce-motion * {
  animation: none !important;
  transition: none !important;
}
```

- JS 动画、Canvas 动画、分数滚动、雷达扫描、点火动画、打字机日志、heartbeat 文案轮换、粒子、`requestAnimationFrame`、长链 `setTimeout` 执行前必须判断 `app.globalData.animationEnabled`。
- reduce-motion 下可以切换静态状态和文字，但不得运行长动画、循环旋转、持续光束、粒子或扫描。
- 所有 timer / interval / animation frame 必须在 `onHide` 或 `onUnload` 清理。
- 动效允许轻微渐入、短扫描线、按钮按压、数值滚动、短点火序列。
- 禁止持续强发光、复杂滚动联动、长时间 loading、无法跳过的打字动画。
- 等待时间超过 6 秒的页面必须提供克制的状态反馈，不能让页面长时间完全静止。

## 前端性能

- 音色试听维护单例 `InnerAudioContext`，执行 `stop() -> src 替换 -> play()`，禁止重复创建实例。
- 页面滚动时不 `setData`，触摸绘图必须节流到一帧一次。
- WXSS 禁止 `transition: all;`，改成明确属性：`transition: opacity .2s, transform .2s, background-color .2s;`。
- Canvas 组件必须在不可见或 reduce-motion 时停止扫描/脉冲动画。
- 普通卡片不用 blur 和阴影；仅主视觉卡、抽屉、浮层允许少量模糊。
- `setData` 只写必要字段，避免整块 `room`、大数组、图表数据在高频事件里反复写入。
- 长等待 heartbeat 文案轮换不得高频 setData，建议 3 秒以上间隔。
- 分享海报 Canvas 应一次性绘制，避免在页面滚动或动画中反复重绘。

## 全息舱页面规范

### 信息层级

全息舱页面分为四层，首屏只展示一级内容：

```text
一级：全息总览 — 镜像投影、协议一致率、航迹样本、全息扫描图
二级：协议行动 — 协议状态卡
三级：协议分析 — 系统判读、信号标签、协议偏移、协议演化（默认折叠）
四级：航迹档案 — 航迹摘要、航迹回放入口
```

协议分析区默认折叠，用户点击「展开详情」后展开。避免首屏堆叠过多分析卡片。

### 操作栏

固定操作栏位于自定义底部导航正上方，包含三个按钮：

操作栏使用 `position: fixed`，`bottom` 基于自定义 tabbar 高度计算（当前为 `calc(140rpx + env(safe-area-inset-bottom))`），`z-index` 低于 tabbar。

页面根容器 `padding-bottom` 必须预留操作栏 + tabbar + safe-area 的总高度。

### 底部安全区

自定义 tabbar 高度为 140rpx + `env(safe-area-inset-bottom)`，`z-index: 999`。

所有使用 `position: fixed` 或 `position: sticky` 的底部元素必须计算 tabbar 高度，避免被遮挡。

校准流程（全屏覆盖层）的 `padding-bottom` 也需包含 `env(safe-area-inset-bottom)`。

### 镜像图片生成

镜像图片使用独立 Canvas 绘制，不是页面截图。海报标识使用「SMART RECORD · HOLO BAY」。

生成过程为一次性绘制，不循环重绘。Canvas 文案必须经过 sanitize。

预览弹窗提供：保存镜像卡、发送给朋友、关闭预览。
